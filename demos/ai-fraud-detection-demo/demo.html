Welcome to our AI Fraud Detection demo

This interactive demo walks a card authorization through Causet’s AI fraud path:

  • You submit a simulated transaction.
  • Causet sends it through an AI fraud decision flow.
  • The AI evaluates risk signals (amount, location, velocity, and more).
  • The result is emitted as a deterministic Causet event.
  • You can inspect the decision, event trail, and outcome.

OpenAI is required — but do not paste your key into this browser. Store it as a Causet secret on the active sandbox fork via the CLI. The demo reads that secret server-side; the browser never displays, stores, or transmits the raw key.

causet secrets set openai --fork sandbox

Configure the key before running the AI fraud demo.

Watch Causet process this transaction

Before we submit this transaction to Causet, open your terminal so you can watch the runtime logs and inspect what changes.

When you continue, Causet will process the fraud decision, emit events, update state, and add entries to the timeline.

After it runs, use the Causet CLI to inspect the fork — state, timeline, and events:

causet context use app ai-fraud-detection causet inspect state --entity txn_9K4 --stream transaction --fork sandbox --application ai-fraud-detection causet inspect timeline --entity txn_9K4 --stream transaction --fork sandbox --application ai-fraud-detection causet inspect event FRAUD_DECISION_MADE

Keep Docker/runtime logs open to watch the intent, AI decision, emitted events, projection updates, and final state change in real time.
