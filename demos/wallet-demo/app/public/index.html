Causet demo

Wallets, ledger & commit envelopes

This app is a thin browser client over Causet: intents write the ledger, projections power the tables, and SSE streams live domain events back to the UI.

How the stack is layered

  1. UI — forms, tables, toasts, activity log
  2. Domain — open / fund / withdraw / transfer orchestration
  3. @causet/sdk — emit, runQuery, SSE streams
  4. Runtime + projections — rules, snapshots, query tables

Try this path

  1. OPEN_WALLET for Alice (and Bob when you transfer)
  2. FUND_WALLET — amounts are integer cents
  3. TRANSFER_START — two-phase commit across source, dest, and ledger
  4. Watch Prepare → Commit → Ledger, balances, and movements update live

What “commit envelope” means

A transfer is not a single row write. Causet runs a prepare / prepared / commit (or abort) cycle so wallets and the ledger stay consistent even if a participant fails. Rejected intents surface the rule code and message in the toast.
